Bildsprache

Fotografie · Illustration · Vier Brand Areas · abgeleitet aus dem Markenrad.

Bilder kommunizieren schneller als Worte. Die Bildsprache von Conciso übersetzt den Kernwert Gelassenheit in visuelle Reize: ruhige Kompositionen, klare Motive und echte Energie statt inszenierter Hochglanz-Ästhetik.

Grundprinzipien aus dem Markenrad

Jeder der drei Markenpfeiler übersetzt sich direkt in visuelle Entscheidungen — von der Wahl des Motivs bis zur Bildbearbeitung.

Stilprinzipien Fotografie

Licht & Ton

Natürliches, weiches Licht · helle, leicht entsättigte Töne · kein Filmdrama, kein HDR · Helligkeit leicht anheben, Kontrast etwas reduzieren

Komposition

Drittel-Regel und goldener Schnitt · Freiflächen für Text-Overlays einplanen · ein dominantes Motiv, ruhige Umgebung · kein Informations-Overload im Bild

Menschen

Authentische Personen aus echten Arbeitskontexten · diverse Repräsentation · Blickkontakt oder echte Konzentration · keine Stockphoto-Posen

Illustration

Illustrationen werden ergänzend eingesetzt — für abstrakte Konzepte, die sich nicht fotografisch abbilden lassen: KI-Prozesse, Systemarchitektur, Organisationsmodelle.

Linienstil

Klare, gleichmäßige Linien · keine Füllflächen ohne Bedeutung · Strichstärke konsistent

Geometrisch

Vereinfachte Formen statt realistischer Darstellung · keine Gradients innerhalb von Figuren

Systemfarben

Ausschließlich Design-System-Tokens · --co-500, --ki-500 etc. · keine Freihandfarben

Weißraum

Leere Fläche ist gestalterisches Element · nicht jeder Pixel muss belegt sein · Illustration atmen lassen

Verwendung

Kontext entscheidet über das richtige Bildformat: Hero-Bereiche brauchen Freiräume für Text, Teaser-Cards arbeiten mit Ausschnitten, und Illustrationen kommen dort zum Einsatz, wo Fotos zu konkret wären.

Kontext Empfehlung
Hero-BereichQuerformat · heller, ruhiger Hintergrund im Bild · untere Bildhälfte frei für Text-Overlay · Foto darf nicht mit der Headline konkurrieren
Card / Teaser16:9 oder 3:2 · Motiv mittig · kein Text im Bild selbst · Brand-Area-Farbe als Eyebrow über dem Bild, nicht als Tint im Bild
WissensartikelIllustration für abstrakte Prozesse · Foto für Praxisbeispiele · Bildunterschrift immer vorhanden · Bilder nie als einziger Informationsträger
TestimonialPorträt im quadratischen oder runden Ausschnitt · Blick zur Kamera oder leicht seitlich · neutraler, heller Hintergrund · kein Hintergrund-Clutter
Barrierefreiheit
AspektRegel
Alt-TextJedes inhaltstragende Bild benötigt einen beschreibenden alt-Text — er beschreibt was zu sehen ist, nicht was das Bild bedeutet; dekorative Bilder erhalten alt=""
Text-OverlayText auf Foto-Hintergründen ≥ 4,5:1 Kontrast einhalten (WCAG 1.4.3) — ggf. dunklen Scrim (rgba(0,0,0,.45)) unter den Text legen; niemals allein auf hellen Bildbereichen vertrauen
Nicht alleinInformationen nie ausschließlich im Bild transportieren — der gleiche Inhalt muss als Text verfügbar sein (WCAG 1.1.1)
BewegungAnimierte Illustrationen oder Hintergrund-Videos mit prefers-reduced-motion pausieren — Nutzerinnen mit Gleichgewichtsstörungen oder Epilepsie sind auf diese Reduktion angewiesen
Dos & Don'ts
✓ Tun
  • ✓Authentische Menschen aus echten Arbeitskontexten wählen — kein gestelltes Stockphoto-Lächeln
  • ✓Helles, leicht entsättigtes Farbklima beibehalten — Gelassenheit überträgt sich auf die Bildstimmung
  • ✓Freiraum für Text-Overlays im Bild einplanen — besonders bei Hero-Fotos
  • ✓Illustrationen ausschließlich mit Design-System-Tokens — Farb-Konsistenz auch im SVG
  • ✓Brand-Area-Farbe als Kontext-Element um das Bild herum einsetzen — nicht als Farbfilter im Bild
✕ Nicht tun
  • ✕Generische Stockfotos (lachende Gruppe am Tisch, Handshake vor weißem Hintergrund) — wirken unecht und austauschbar
  • ✕Übersättigte oder hochkontrastige Bearbeitungen — widersprechen dem Markenwert Gelassenheit
  • ✕Bereichsfarbe als Tint-Filter über das Foto legen — wirkt aufdringlich, überdeckt die natürliche Bildstimmung
  • ✕Realistische oder fotografische Illustrationsstile — bricht den klaren, geometrischen Linienstil
  • ✕Bilder ohne alt-Text oder mit generischem „Foto" — Screenreader-Nutzende erhalten keinen Mehrwert

Dos & Don'ts
✓ Tun
  • ✓Libre Baskerville für Überschriften und emotionale Akzente — der markentypische Kontrast zur UI
  • ✓Display-Tokens ausschließlich auf Landing Pages und Kampagnenseiten einsetzen
  • ✓Body Large (16px) als Standard für Fließtext — optimale Lesbarkeit auf allen Screens
  • ✓Token-Hierarchie einhalten: Display → Headline → Title → Body → Label
  • ✓Label-Tokens für alle UI-Beschriftungen: Buttons, Chips, Badges, Eyebrows
✕ Nicht tun
  • ✕Libre Baskerville als Fließtext — Serifenschriften ermüden bei langen Texten auf Screens

Wann die farbige Linie oben?

Jede Karte gehört zu einer Brand Area — das muss auf einen Blick erkennbar sein. Wie das Signal transportiert wird, hängt davon ab, ob die Karte einen Media-Bereich hat oder nicht:

Mit Media-Bereich (Generische Karten: Elevated, Filled, Outlined) — die großflächige Fläche in der Bereichsfarbe -50 trägt das Farbsignal bereits. Eine zusätzliche Linie wäre redundant und würde das ruhige Erscheinungsbild stören.
Ohne Media-Bereich (Inhaltskarten: Testimonial, Stat, Download CTA) — die Karte besteht nur aus Text und Aktionen. Ohne die 4 px Linie oben fehlt jeder visuelle Hinweis auf die Zugehörigkeit. Der Top-Border ist dann das einzige Farbsignal.

Warum oben und nicht links? Die Leserichtung ist von links nach rechts, von oben nach unten — der oberste Kartenrand ist der erste Fixpunkt beim Scannen einer Seite. Die Linie links eignet sich für redaktionellen Fließinhalt (Blockquote), weil der Text direkt daneben beginnt und die Linie als Begleiter wirkt. Bei einer abgeschlossenen Karte würde ein linker Rand mit dem Layout-Raster kollidieren und die Außenkante der Karte unruhig machen.

Inline-Code

Für Code-Erwähnungen innerhalb von Fließtext. Klasse .cb-prose am umgebenden Element aktiviert das Styling für alle <code>-Elemente darin.

Um die Hintergrundfarbe einer Seite zu ändern, überschreibe den Token --bg-page innerhalb eines [data-theme]-Selektors. Der Token --bg-surface sollte dabei immer einen leicht höheren Kontrast zu --bg-page bieten, damit Karten und Panels sichtbar bleiben.

ARIA Patterns

Focus-Ring

Sichtbarer Fokusindikator für alle interaktiven Elemente (WCAG 2.4.7).

--focus-ring: 0 0 0 3px var(--co-50), 0 0 0 5px var(--co-500); :focus-visible { outline: none; box-shadow: var(--focus-ring); }

Formulare & Fehler

Fehlermeldungen programmatisch mit dem Feld verknüpfen (WCAG 1.3.1, 3.3.1).

<input aria-invalid="true" aria-describedby="err-id"> <span id="err-id" role="alert"> Pflichtfeld </span>

Dekorative Icons

SVGs ohne inhaltliche Bedeutung vor Screenreadern verbergen (WCAG 1.1.1).

<svg aria-hidden="true" focusable="false" viewBox="0 0 24 24"> <path d="…"/> </svg>

Icon-only Buttons

Buttons ohne sichtbares Label brauchen ein zugängliches Textalternativ (WCAG 1.1.1).

<button aria-label="Menü öffnen"> <svg aria-hidden="true" focusable="false">…</svg> </button>

Tab Panels

Tabs als Tablist mit vollständigen ARIA-Relationen (WCAG 4.1.2).

<div role="tablist"> <button role="tab" aria-selected="true" aria-controls="panel-1" id="tab-1">Tab 1</button> </div> <div role="tabpanel" id="panel-1" aria-labelledby="tab-1" tabindex="0">…</div>

Live Regions

Dynamische Inhalte für Screenreader ankündigen (WCAG 4.1.3).

/* Status (höflich, unterbricht nicht) */ <div role="status" aria-live="polite"> Gespeichert. </div> /* Alert (sofort, unterbricht) */ <div role="alert" aria-live="assertive"> Verbindungsfehler. </div>

Touch Targets & Interaktionsflächen

Interaktive Elemente müssen mindestens 44 × 44 CSS-Pixel groß sein (WCAG 2.5.5 Level AAA / 2.5.8 Level AA in WCAG 2.2).

✓
Buttons & Links: min. 44 × 44px via Padding erreichen, auch wenn der sichtbare Bereich kleiner ist.
✓
Icon-Buttons: min. 48px empfohlen, sichtbares Icon kann kleiner sein.
✗
Kleine Chips/Badges: Nicht klickbar machen ohne ausreichende Trefferfläche. Padding erhöhen oder Wrapper verwenden.

Mindestgröße per CSS

.btn { min-height: 44px; min-width: 44px; padding: 10px 20px; } /* Icon-only: Fläche über padding */ .icon-btn { width: 48px; height: 48px; display: flex; align-items: center; justify-content: center; }

Reduzierte Bewegung & High Contrast

prefers-reduced-motion

Nutzer mit Gleichgewichts- oder Aufmerksamkeitsstörungen können Animationen deaktivieren (WCAG 2.3.3 AAA).

@media (prefers-reduced-motion: reduce) { *, *::before, *::after { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; scroll-behavior: auto !important; } }

forced-colors (High Contrast)

Windows High Contrast Mode ersetzt Farben durch Systemfarben. Grenzen durch Borders statt nur Farbe ausdrücken.

@media (forced-colors: active) { .btn-filled { /* Border sichert Sichtbarkeit */ border: 2px solid ButtonText; forced-color-adjust: none; } .btn-outlined { border-color: ButtonText; } }

prefers-color-scheme

Dark Mode folgt der Betriebssystem-Präferenz. Alle Farb-Tokens werden über [data-theme="dark"] überschrieben.

@media (prefers-color-scheme: dark) { /* Automatisch anwenden: */ :root { color-scheme: dark; } } /* Oder manuell per JS: */ document.documentElement .dataset.theme = 'dark';
